## Overview

Start building your documentation with learn Documentation today. Follow these steps to create an account, set up your space, add your first page, and navigate effectively. You complete the process in under five minutes.

<Callout kind="tip">
  Use your brand color `#3B82F6` in custom themes for a consistent look.
</Callout>

## Create an Account

Sign up quickly to access your documentation space.

<Steps>
  <Step title="Visit the Dashboard" icon="globe">
    Go to the [learn Documentation homepage](https://learn-docs.com) and click **Sign Up**.
  </Step>
  <Step title="Enter Details" icon="user">
    Provide your email, create a password, and verify your account via email.
  </Step>
  <Step title="Complete Setup" icon="check-circle">
    Choose a workspace name like `my-project-docs`.
  </Step>
</Steps>

## Set Up Your Documentation Space

Configure your space for your project needs.

<Tabs>
  <Tab title="Web UI" icon="browser">
    From the dashboard, select **New Space** and enter details:
    
    - Name: `API Reference`
    - Description: `Documentation for our REST API`
    
    Enable features like search and versioning.
  </Tab>
  <Tab title="CLI" icon="terminal">
    Install the CLI and initialize:
    
    ```bash
    npm install -g @learn/docs-cli
    learn-docs init my-space
    ```
    
    Edit `learn-docs.json`:
    
    ```json
    {
      "name": "My Space",
      "brandColor": "#3B82F6"
    }
    ```
  </Tab>
</Tabs>

## Add Your First Page

Create a `quickstart.mdx` file in your space root.

<CodeGroup tabs="MDX,YAML">
  ```mdx
  ---
  title: Welcome
  description: Your first documentation page.
  ---
  
  ## Hello World
  
  Start writing your docs here.
  ```
  ```yaml
  # frontmatter.yaml
  title: Welcome
  description: Your first page.
  ```
</CodeGroup>

Preview and publish from the editor. Use the `{title}` variable in templates for dynamic content.

<Callout kind="info">
  Always include YAML frontmatter for proper rendering. Avoid blank lines inside it.
</Callout>
